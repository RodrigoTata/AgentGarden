# Debug Report Formats

This document defines the exact formatting for the reports generated by the `debug` skill. All reports must be presented in Spanish.

---

## Initial Triage (Step 1)

```text
🔍 DIAGNÓSTICO INICIAL
━━━━━━━━━━━━━━━━━━━━━━
Problema:    [Restate what the user described, in their own words]
Nivel:       [🟢/🟡/🟠/🔴] T[N] — [Name]
Mi teoría:   [One-sentence hypothesis in simple language]
Plan:        [What you're going to do next, explained simply]
━━━━━━━━━━━━━━━━━━━━━━
```

---

## T2: Logic Bug Report (Step 2)

```text
🔍 CAUSA ENCONTRADA (T2)
━━━━━━━━━━━━━━━━━━━━━━━━
Lo que ves:      [What the user sees, in their words]
Lo que pasa:     [Simple explanation of what went wrong — no jargon]
Cómo lo arreglo: [What will change, explained simply]
Archivos:        [List of files to modify]
━━━━━━━━━━━━━━━━━━━━━━━━
```

---

## T3: Structural Issue Report (Step 2)

```text
🔍 ÁRBOL DE FALLA (T3)
━━━━━━━━━━━━━━━━━━━━━━
Lo que reportas:  [User's description]

Camino del error:
  [User action] → [Frontend] → [Backend endpoint] → 💥 [Where it breaks]

Descartados:
  ✗ [Thing you checked and ruled out — in simple terms]
  ✗ [Another ruled-out cause]

Causa raíz:
  ✓ [The actual issue — simple explanation + technical detail]

Plan de fix:
  1. [First change — what it fixes in user terms]
  2. [Second change]
  N. [Cómo lo verificamos juntos]
━━━━━━━━━━━━━━━━━━━━━━
```

---

## T4: Systemic Issue Report (Step 2)

```text
🔍 ANÁLISIS DE CAUSA RAÍZ (T4)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Resumen para el usuario:
  [2-3 sentences explaining the problem in plain language]

Timeline:
  - [Date/Event] [What happened]
  - [Date/Event] [What changed]

Alcance del daño:
  - [Feature/area affected — in user-facing terms]
  - [Data impact — what data might be wrong]

Hipótesis:
  H1: [Hypothesis] → [Evidence] → [DESCARTADA/CONFIRMADA]
  H2: [Hypothesis] → [Evidence] → [DESCARTADA/CONFIRMADA]

Causa raíz:
  [Confirmed root cause — simple explanation first, then technical detail]

Plan de fix:
  1. [Fix urgente — detener el problema]
  2. [Fix estructural — que no vuelva a pasar]
  3. [Verificación — qué debe probar el usuario]
  4. [Prevención — cómo evitamos esto en el futuro]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```
